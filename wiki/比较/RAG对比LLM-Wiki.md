# RAG 对比 LLM Wiki

> 此页面是概念页面 [[RAG对比LLM-Wiki]] 的比较扩展版

## 概述

RAG 和 LLM Wiki 代表了两种根本不同的知识利用范式。前者是"即时检索"，后者是"增量编译"。

## 逐维度对比

### 知识存储

| | RAG | LLM Wiki |
|---|---|---|
| 存储形式 | 原始文档 + 向量索引 | 结构化 Markdown + 交叉引用 |
| 知识状态 | 静态（索引不变） | 动态（持续更新） |
| 知识密度 | 低（原文冗余多） | 高（已提炼综合） |

### 查询过程

| | RAG | LLM Wiki |
|---|---|---|
| 第一步 | 向量搜索 top-k 片段 | 搜索相关 wiki 页面 |
| 第二步 | 拼凑片段到 prompt | 读取已综合的页面 |
| 第三步 | LLM 从片段生成答案 | LLM 从综合生成答案 |
| 质量上限 | 受限于检索窗口 | 受限于 wiki 覆盖度 |

### 长期价值

| | RAG | LLM Wiki |
|---|---|---|
| 第 1 天 | 中等 | 低（wiki 未建立） |
| 第 30 天 | 中等（不变） | 高（wiki 已丰富） |
| 第 365 天 | 中等（仍不变） | 很高（wiki 高度综合） |
| 趋势 | 平坦 | 指数增长 |

### 适合场景

| 场景 | RAG ✅ | LLM Wiki ✅ |
|------|--------|-------------|
| 一次性文档问答 | ✅ | — |
| 快速原型 | ✅ | — |
| 长期研究项目 | — | ✅ |
| 竞品分析 | — | ✅ |
| 读书笔记 | — | ✅ |
| 个人知识管理 | — | ✅ |
| 企业内部知识库 | 部分 | ✅ |

## 混合策略

RAG 和 LLM Wiki 不完全互斥：
- 初始阶段可用 RAG 快速启动
- 逐步用摄取操作将 RAG 覆盖的来源编译进 wiki
- 最终 wiki 足够丰富时，RAG 退化为补充手段
