# Agent设备化

## 是什么

Agent 设备化是指 AI Agent 不再仅存在于对话界面中，而是拥有独立的硬件设备（云手机、云电脑），可以在真实操作系统上自主操作，像人类一样浏览网页、运行程序、处理文件。

## 为什么重要

这是 Agent 从"对话助手"走向"数字员工"的关键一步：

- **突破对话框边界**：Agent 不再只能"聊"，而是能"做"
- **操作真实世界**：下载 APP、搜索内容、运行代码脚本——这些都需要设备支撑
- **人机协作新模式**：复杂操作由 Agent 执行，关键决策由人类接管

## 当前实现

以 [[扣子]] 2.5 为例：

| 设备 | 能力 | 限制 |
|------|------|------|
| 云电脑 | 运行代码脚本、浏览网页、处理文件 | 部分网站被拦截 |
| 云手机 | Android 13 系统，下载 APP、滑动页面、点击操作 | 登录/引导页易卡住，按钮识别不准 |

## 核心挑战

1. **环境适应性差**：登录弹窗、引导页、验证码等真实环境要素频繁导致任务中断
2. **操作精度不足**：点击位置识别、文字提取等底层操作仍不稳定
3. **人机切换不流畅**：何时需要用户接管、如何平滑切换尚未解决
4. **进度不透明**：用户难以了解 Agent 的执行进度，追问时只能得到"快了"之类的模糊回应

## 发展方向

- **短期**：提升特定 APP 的操作适配，减少卡顿
- **中期**：优化人机协作流程，让接管与交还更自然
- **长期**：Agent 能真正独立完成端到端任务，人类只需审核结果
