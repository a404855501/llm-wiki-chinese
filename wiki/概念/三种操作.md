# 三种操作

## 是什么

llm-wiki 定义了三种核心操作，覆盖知识库的完整生命周期：摄取新知识、查询已有知识、维护知识健康。

## 摄取（Ingest）

**触发**：用户将新来源放入 `sources/` 并要求 LLM 处理。

**工作流**：
1. LLM 完整阅读原始来源
2. 与用户讨论关键要点
3. 在 wiki 中创建摘要页面
4. 更新 `index.md`
5. 更新所有相关的实体和概念页面
6. 在 `log.md` 追加条目

**影响范围**：单个来源可能触及 10–15 个 wiki 页面。

**关键**：不是索引，而是**整合**。新知识必须融入已有结构。

## 查询（Query）

**触发**：用户针对 wiki 提问。

**工作流**：
1. LLM 搜索相关 wiki 页面
2. 阅读并综合
3. 生成带引用的答案

**输出形式**：
- Markdown 页面
- 比较表格
- 幻灯片（[[Marp]]）
- 图表（matplotlib / plotly）
- 交互式 Canvas

**重要洞察**：**好的答案可以归档回 wiki 成为新页面**——查询本身也是知识积累。

## 健康检查（Lint）

**触发**：用户定期要求 LLM 检查 wiki 健康状况。

**检查项**：
- [ ] 页面之间的矛盾
- [ ] 被新来源取代的过时声明
- [ ] 没有入站链接的孤立页面
- [ ] 被提及但缺少页面的重要概念（红链）
- [ ] 缺失的交叉引用
- [ ] 可通过网络搜索填补的数据空白

**类比**：就像代码的 lint 检查——不是改功能，而是保持代码库健康。

## 操作频率

| 操作 | 频率 | 类比 |
|------|------|------|
| 摄取 | 每次添加新来源 | git commit |
| 查询 | 日常使用 | git log/diff |
| 健康检查 | 定期（周/月） | CI/CD 检查 |
