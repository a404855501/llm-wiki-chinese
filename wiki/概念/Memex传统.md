# Memex 传统

## 是什么

llm-wiki 在精神上延续了范内瓦·布什（Vannevar Bush）1945 年提出的 Memex 构想——一个个人策划的知识库，文档之间有关联的痕迹（Associative Trails）。

## 范内瓦·布什的 Memex (1945)

布什在《As We May Think》中设想了一种设备：
- 个人可以存储所有书籍、记录和通信
- 具有高速和灵活性，可供随时查阅
- 最重要的是：文档之间的**关联痕迹**与文档本身一样有价值
- 私有的、积极策划的

## 从 Memex 到 LLM Wiki

| 维度 | Memex (1945) | LLM Wiki (2026) |
|------|-------------|-----------------|
| 存储 | 微缩胶片 | Markdown 文件 |
| 关联 | 人工编码的痕迹 | LLM 维护的双链 |
| 策划 | 人类手动 | 人类 + LLM 协作 |
| 维护 | **未解决** | **LLM 承担** |
| 可扩展性 | 受限于物理介质 | 受限于 LLM 上下文窗口 |

## 布什未能解决的问题

布什构想中最核心的缺口：**谁来做维护工作？**

人类放弃 wiki 是因为维护负担增长快于价值：
- 更新交叉引用很繁琐
- 保持摘要最新需要持续投入
- 注意矛盾需要同时记住所有内容

LLM 解决了这个问题：
- 不会感到无聊
- 不会忘记更新交叉引用
- 可以一次触及 15 个文件

## 思想谱系

```
Memex (1945, 范内瓦·布什)
  → 超文本 (1960s, 泰德·纳尔逊)
    → 万维网 (1989, 蒂姆·伯纳斯-李)
      → Wiki (1995, 沃德·坎宁安)
        → 个人知识管理 (Zettelkasten, Roam, Obsidian)
          → LLM Wiki (2026, 安德烈·卡帕西)
```
