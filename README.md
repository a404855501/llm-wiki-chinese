# llm-wiki 中文版

> 基于 [Andrej Karpathy 的 llm-wiki Gist](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) 搭建的中文知识库

## 这是什么

llm-wiki 是一种使用 LLM **增量构建和维护持久知识库**的模式。与传统 RAG（每次查询都从原始文档重新检索）不同，llm-wiki 让 LLM 持续编译知识到一个结构化的 wiki 中——交叉引用预建、矛盾预标记、综合持续演进。

本仓库是 Karpathy 原始 Gist 的**完整中文汉化版**，包括目录结构、文件名、frontmatter 字段和所有正文内容均已中文化。

## 目录结构

```
llm-wiki/
├── SCHEMA.md              ← LLM 维护规范（核心配置）
├── index.md               ← 内容目录
├── log.md                 ← 操作日志（只追加）
├── sources/               ← 原始来源层（不可变）
│   └── karpathy-llm-wiki-gist.md
└── wiki/                  ← Wiki 层（LLM 读写）
    ├── 概念/              ← 抽象思想、方法论、模式（6 页面）
    ├── 实体/              ← 人物、工具、项目（8 页面）
    ├── 比较/              ← A 与 B 对比分析（1 页面）
    └── 概览/              ← 主题综合、领域地图（2 页面）
```

## 核心概念

| 概念 | 说明 |
|------|------|
| 增量知识编译 | 知识编译一次、持续更新，而非每次查询重新派生 |
| 三层架构 | 原始来源 → Wiki → 模式，权限分明 |
| 三种操作 | 摄取（Ingest）/ 查询（Query）/ 健康检查（Lint） |
| 持久复合产物 | Wiki 随使用增值，交叉引用预建、矛盾预标记 |
| RAG 对比 LLM Wiki | 即时检索 vs 增量编译：两种知识利用范式 |
| Memex 传统 | 从布什 (1945) 到卡帕西 (2026) 的思想谱系 |

## 如何使用

### 1. 浏览

用 [Obsidian](https://obsidian.md) 打开本仓库文件夹，即可获得双链跳转和图形视图。

### 2. 添加来源

将新文档放入 `sources/` 目录，然后让 LLM 执行**摄取**操作——读取来源、创建摘要、更新相关页面、追加日志。

### 3. 提问

针对 wiki 内容提问，LLM 会搜索相关页面并综合回答。好的答案可以归档回 wiki 成为新页面。

### 4. 健康检查

定期让 LLM 对 wiki 做**健康检查**——发现矛盾、孤立页面、缺失引用等。

## 汉化约定

| 类别 | 规则 | 示例 |
|------|------|------|
| 目录名 | 中文 | `概念/`、`实体/`、`比较/`、`概览/` |
| 文件名 | 中文，专有名词保留原文 | `增量知识编译.md`、`Obsidian.md` |
| Frontmatter 字段 | 中文 | `标题`、`类型`、`创建日期`、`状态` |
| 状态值 | 中文 | `稳定`、`草稿`、`待审核` |
